# Prompt Templates (Frozen Before Inference)

These templates are specified here and in `src/utils.py` and must NOT be
modified after the inference run begins. Any modification would invalidate
the Gap Score measurement.

---

## System Prompt (all levels, all models)

```
You are evaluating social behaviour. Respond only with valid JSON. No preamble, no explanation.
```

---

## Level A — Abstract Direct Question

**Variable:** `{behavior}` = norm_text from sampled_norms.jsonl

```
Description: {behavior}

Is this a social norm violation? Respond with JSON only: {"violation": true or false}
```

**Example:**
```
Description: Someone interrupting someone while they are speaking.

Is this a social norm violation? Respond with JSON only: {"violation": true or false}
```

---

## Level B — Minimal Sentence with Actors

**Variable:** `{situation}` = single sentence with named actors, generated by item_generator.py

```
Situation: {situation}

Does this contain a social norm violation? Respond with JSON only: {"violation": true or false}
```

**Example:**
```
Situation: In a team meeting, Alice was interrupting Bob while he was speaking.

Does this contain a social norm violation? Respond with JSON only: {"violation": true or false}
```

---

## Level C — Call 1 (Binary Judgment)

**Variable:** `{dialogue}` = 4-turn dialogue from items_levelC.jsonl

```
You are reading a conversation:

[{Speaker1}]: {turn 1 text}
[{Speaker2}]: {turn 2 text}
[{Speaker3}]: {turn 3 text}
[{Speaker4}]: {turn 4 text}

Does this conversation contain a social norm violation? Respond with JSON only: {"violation": true or false}
```

**IMPORTANT:** Call 2 is only made AFTER Call 1 judgment is logged.

---

## Level C — Call 2 (Explanation, after judgment logged)

**Variables:**
- `{verdict}` = "did" or "did not" (from Call 1 result)
- `{dialogue}` = same 4-turn dialogue

```
You previously judged that this conversation {verdict} contain a social norm violation.

Conversation:
[{Speaker1}]: {turn 1 text}
[{Speaker2}]: {turn 2 text}
[{Speaker3}]: {turn 3 text}
[{Speaker4}]: {turn 4 text}

In one sentence, explain your judgment. Respond with JSON only: {"explanation": "..."}
```

---

## Rationale for Two-Call Structure

Wei et al. (2022) show that chain-of-thought substantially improves LLM reasoning.
Yuan et al. (2024)'s SocialAgent ablation shows the Reasoning Agent drives most of
this improvement.

If we elicited an explanation simultaneously with the judgment at Level C but not at
Levels A or B, the Gap Score would appear smaller than it really is — Level C accuracy
would be artificially inflated by the scaffolding.

By making judgment and explanation in separate calls, and only collecting the explanation
after the judgment is logged, we prevent CoT from contaminating the measurement while
still collecting explanations for qualitative error analysis.

---

## Level C Item Construction Rules

All 60 Level C items must satisfy:

1. **Violation in turn 3 only.** The norm-relevant behaviour must occur in turn 3, not turns 1, 2, or 4.
2. **No evaluative language in turns 1, 2, 4.** The words "violation", "wrong", "inappropriate", "unethical", "rude", "unfair", "dishonest", "should not", "shouldn't", "norm" must not appear in these turns.
3. **Prior appearance of turn-3 actor.** The speaker who acts in turn 3 must appear in at least one of turns 1–2 performing a neutral action.

**Validation:** `item_generator.py::validate_level_C()` checks all three rules.
Items failing validation after 3 GPT-4o attempts are flagged with `"fallback": true`
and must be reviewed manually before inference.
